***
*** Exercice STUDI: Créer une application de gestion de listes ***
***

*** Etude du besoin et fonctionnalités identifiées:

1) Application WEB
2) Possible de Creer, Mettre a jour, Supprimer, Modifier (CRUD)
	des listes 	{
		un nom
		une ou plusieurs tâches
		suivi de la progression d'un simple coup d'oeil (barre de progression ou numero effectué/total
			}
	des tâches 	{
		un nom
		un statut modifiable (a faire, en cours, terminée, en suspend)
		un créateur
		date de création
		un responsable
			} 
3) utilisation en interne dans le cadre de la gestion de projets
	--> collaboratif // en équipe

4) faire seulement la gestion de la liste de tâches

5) Fonctionnalité supplémentaire:
	classement des tâches par catégorie:
		Matrice Eisenhower: Urgent & Important -- Urgent & Pas Important -- Pas Urgent & Important -- Pas Urgent & Pas Important
		Par date de fin

8) Idées vracs:
 - avoir un affichage des listes par personnes: Moi // Pierre // Jean
 - Avoir une date butoire pour la réalisation de la tâche
 - avoir une alerte pour les tâches urgentes
***

*** Langages utilisés:
 - Front: HTML, CSS, JS --> application web, langages connus et souvent utilisés
 - Back: PHP --> s'intègre très bien avec le HTML, facile à utiliser avec Mysql.
 - BDD: mysql --> BDD relationnelle

Autres outils:
 - Git -> gestion des versions de code
 - VS code -> environnement de programmation prenant en compte différents langages
 - Hebergeur: Hostinger --> dispose d'une BDD mysql et permet l'edition des fichiers en direct (outil de gestion des fichiers propriétaire).
